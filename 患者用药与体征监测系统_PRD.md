# 患者用药与体征监测系统 — 产品需求文档 (PRD)

版本：v1.0 | 创建日期：2026-06-28

---

## 1. 产品概述

### 1.1 产品定位
一款面向家庭照护场景的 PWA（渐进式 Web 应用），帮助照护者管理多位患者的日常体征记录、用药方案与提醒，支持数据导出分享给医生。

### 1.2 核心目标
- 降低照护者的记忆负担，避免漏服、漏测
- 提供直观的体征趋势，辅助健康决策
- 便捷的数据导出能力，支撑医患沟通

### 1.3 用户群体
- 主要用户：家庭照护者（子女照护老人、配偶照护等）
- 覆盖患者数：≤10 人

---

## 2. 功能模块

### 2.1 患者管理模块

**功能描述**：管理患者档案，作为体征与用药记录的核心主体。

**输入信息**：
- 基础信息：姓名、性别、年龄、联系方式（选填）
- 健康档案：基础病（多选标签 + 自定义输入）、过敏史
- 备注：长文本，用于记录特殊护理注意事项

**展示方式**：
- 患者列表页：卡片式展示，高亮显示"今日未记录"或"有用药提醒"状态
- 患者详情页：Tab 切换"体征记录"、"用药方案"、"个人信息"

**关键规则**：
- 支持 ≤10 人管理，超出提示归档
- 删除患者需二次确认，关联数据一并删除
- 基础病支持历史输入记忆

### 2.2 每日体征记录模块

**功能描述**：按日记录关键生命体征，支持趋势可视化。

**输入信息**：
- 日期：默认当天，支持补录过去 7 天
- 数值：收缩压/舒张压（mmHg）、心跳（bpm）、体重（kg）
- 备注：当日特殊情况（如"头晕"）

**展示方式**：
- 录入页：大键盘数字输入，异常值红色高亮预警（收缩压 > 180）
- 趋势图：折线图展示近 7 天/30 天变化，支持长按查看备注

**关键规则**：
- 每日仅一条主记录，重复录入提示"覆盖"或"追加备注"
- 数值范围校验：心跳 30-220，体重 20-300，超范围需二次确认
- 首页"今日体征"卡片未录入时显示"待记录"状态

### 2.3 用药管理与提醒模块

**功能描述**：建立药品库，配置用药方案，提供每日用药看板与提醒。

**输入信息**：
- 药品库：药品名称、规格、分类（降压/降糖/保健）、备注
- 用药方案：关联药品、单次剂量、频次、具体时间点、起止日期

**展示方式**：
- 今日用药看板：时间轴布局，显示"待服"、"已服"、"跳过"状态，支持一键打卡
- 提醒通知：PWA 系统通知 + 应用内 Toast 提示

**关键规则**：
- 提醒依赖 Notification API，前台轮询、后台系统通知（需授权）
- 首页必须常驻"今日待服"列表作为兜底
- 打卡自动扣减虚拟库存，库存 < 3 日用量时显示"需续方"标签
- 支持一键暂停/恢复用药方案

### 2.4 数据导出模块

**功能描述**：将本地数据打包导出，用于备份或分享给医生。

**输入信息**：
- 导出范围：全部数据 / 指定患者 / 指定时间段
- 导出格式：Excel（.xlsx）或 JSON（.json）

**展示方式**：
- 导出预览：显示记录条数、文件大小
- 下载触发：浏览器下载管理器，文件名格式 `患者姓名_健康数据_YYYYMMDD.xlsx`

**格式规范**：
- Excel：Sheet1 患者信息 → Sheet2 体征记录 → Sheet3 用药记录
- JSON：结构化数据，含 schemaVersion 字段，支持未来导入恢复
- 安全提示：文件不含密码，提示用户妥善保管

## 3. 非功能需求

| 类别 | 要求 |
|------|------|
| 存储 | 全部数据存储于浏览器本地（IndexedDB / localStorage），无需服务端 |
| 离线可用 | 核心功能（浏览、录入、打卡）完全离线可用 |
| 响应式 | 适配移动端为主、桌面端可用 |
| 通知 | 依赖浏览器 Notification API，需用户授权 |
| 兼容性 | 支持最新版 Chrome / Edge / Safari |

---

## 4. 隐私与安全

- 所有数据仅存储在用户设备本地，不上传任何云端
- 导出文件不含密码保护，需用户自行妥善保管
- 删除患者时永久清除所有关联数据

---

## 5. 约束与限制

- 患者总数上限 10 人
- 体征记录仅支持补录过去 7 天
- 提醒功能受浏览器策略限制，后台不可靠时由首页看板兜底
