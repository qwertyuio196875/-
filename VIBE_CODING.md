# 患者用药与体征监测系统 — Vibe Coding 开发流程

基于 AI 辅助的对话式开发方案，14 天交付可用 PWA。

---

## 核心理念

用自然语言描述需求 → AI 生成代码 → 人工确认验收 → 迭代下一块。开发者专注于"做什么"和"对不对"，AI 负责"怎么写"。

---

## 阶段一：项目脚手架（第 1 天）

| 步骤 | 操作 |
|---|---|
| 1 | 让 AI 初始化 Vite + Vue 3 + TS 项目，安装 Vant 4、Dexie.js、vite-plugin-pwa |
| 2 | 让 AI 生成 AGENT.md 中的三个数据表（Medicine、DoseRecord、VitalsRecord）的 Dexie 数据库定义 |
| 3 | 让 AI 配置 PWA manifest + Service Worker 基础模板 |
| 4 | 确认：`npm run dev` 能正常启动，页面可访问 |

**示例 Prompt**：

> 初始化一个 Vite + Vue 3 + TypeScript 项目，安装 Vant 4、Dexie.js、vite-plugin-pwa。创建 src/db/index.ts，用 Dexie 定义 Medicine、DoseRecord、VitalsRecord 三个表，字段见 AGENT.md 数据模型章节。

**示例 Prompt**：

## 阶段五：数据导入导出（第 9-10 天）

| 步骤 | 操作 |
|---|---|
| 1 | 让 AI 实现 JSON 导出（含 schemaVersion + AES-GCM 加密） |
| 2 | 让 AI 实现 JSON 导入（解密 → 校验 → 冲突处理 UI） |
| 3 | 让 AI 实现 Excel 导出（SheetJS，三张 Sheet） |
| 4 | 确认：导出文件能正常解密导入、Excel 格式正确 |

**示例 Prompt**：

> 参照 PRD 2.4 数据导出模块和 TECH_DESIGN 3.2 加密导出流程。1）导出：序列化 IndexedDB 数据 → 用户输入密码 → PBKDF2 派生密钥 → AES-GCM 加密 → 生成 .med.json 下载。2）导入：读取文件 → 解密 → 校验 schemaVersion → 弹 ActionSheet 让用户选覆盖/跳过/合并 → 写入 DB。3）Excel 导出用 SheetJS，Sheet1 患者信息 / Sheet2 体征记录 / Sheet3 用药记录。

## 开发原则

- **一次一个模块**：完成一个模块并验证通过后，再进入下一个
- **AI 生成后必须手动检查**：重点看数据流是否正确、边界情况是否处理
- **及时问 AI**：遇到不理解的地方直接问，把 AI 当作结对编程伙伴
- **小步提交**：每完成一个子功能就 git commit，方便回溯
- **用 AGENT.md 喂 AI**：每次开启新对话时，先把 AGENT.md 发给 AI 建立上下文
